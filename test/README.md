# Тестирование CRUD MongoDB Server

Этот проект использует встроенный в Node.js тестовый фреймворк `node:test` для тестирования сервера с мокированной MongoDB.

## Структура тестов

- `setup.ts` - Настройка тестового окружения с моками
- `mocks/mongodb.js` - Моки для MongoDB клиента и операций
- `utils/httpClient.ts` - Утилиты для HTTP тестирования
- `server.test.ts` - Базовые тесты HTTP сервера (CORS, OPTIONS, обработка ошибок)
- `helpDesk.test.ts` - **Интеграционные тесты Help-Desk API**
- `likeATrello.test.ts` - **Интеграционные тесты Like-A-Trello API**
- `crud.test.ts` - Юнит-тесты для сложной бизнес-логики CRUD операций
- `validation.test.ts` - Тесты валидации запросов

## Запуск тестов

### Запуск всех тестов
```bash
yarn test
```

### Запуск тестов в режиме watch
```bash
yarn test:watch
```

## Мокирование MongoDB

Тесты используют полностью мокированную MongoDB, что позволяет запускать их в CI/CD средах без необходимости установки реальной базы данных.

### Преимущества моков:
- ✅ Работают в GitHub Actions и других CI/CD системах
- ✅ Быстрое выполнение тестов
- ✅ Изолированная среда для каждого теста
- ✅ Нет зависимостей от внешних сервисов

### Реализованные моки:
- **MockMongoClient** - Имитирует MongoClient
- **MockDb** - Имитирует базу данных
- **MockCollection** - Имитирует коллекцию с основными CRUD операциями

## Архитектура тестирования

### Разделение ответственности
- **Интеграционные тесты** проверяют полный цикл: HTTP запрос → обработка → ответ
- **Юнит-тесты** проверяют сложную бизнес-логику в изоляции
- **Моки** обеспечивают быстрое и надежное тестирование без внешних зависимостей

### HTTP утилиты
`utils/httpClient.ts` предоставляет:
- **HttpClient** - удобный интерфейс для HTTP запросов
- **assertJsonResponse** - проверка успешных JSON ответов
- **assertErrorResponse** - проверка ответов с ошибками
